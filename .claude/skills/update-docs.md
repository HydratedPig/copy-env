# 文档更新指令

当源代码发生变更时，你需要自动更新项目的多语言文档，确保所有文档版本保持一致性和准确性。

## 工作流程

### 1. 分析代码变更
使用 `git diff` 或读取相关源文件，识别变更类型：
- 新增功能：添加完整的功能说明和示例
- 修改功能：更新相关的 API 文档和使用说明
- 配置变更：更新配置选项说明
- Bug 修复：更新已知问题或使用注意事项

### 2. 更新文档
按以下顺序更新文档，保持各语言版本的内容结构一致：
1. README.md（英文主文档）
2. docs/README.zh-CN.md（简体中文）
3. docs/README.zh-TW.md（繁体中文）
4. docs/README.ja.md（日文）

### 3. 确保质量
- 代码示例必须可运行且与实际代码一致
- 配置示例必须准确反映当前 API
- Markdown 格式正确
- 保持文档风格一致
